# client-serverApp  

Разработки клиент-серверного приложения, использующих стандартные механизмы межпроцессного взаимодействия.  

В этой работе было реализовано 2 сервера и клиент, при помощи сокетов было налажено их взаимодействие. Клиент использует интерфейс для получения информации, которую предоставляет первый и второй сервер. Помимо этого был реализован логгер, который отслеживает и записывает каждое взаимодействия пользователя в отдельный файл.

Также реализованы доп задания:
1.	При периодическом обновлении информации по запросу от клиента (организовать выбор режима отправки периодических запросов к серверу в рамках текущей сессии) новые данные передается сервером в ответ только в том случае, если что-либо изменилось с момента прошлого обновления информации для данного клиента.
2.	Запуск клиентов в контейнерах (docker).
3.	Реализация графического интерфейса на стороне клиента.
